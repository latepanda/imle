Mercury library

Mercury

A modular, mobile-first class-light library for semantic HTML by Ellie.

Getting started

Page setup

Import Mercury to get styled semantic HTML first, then append utility and component classes only where the layout needs more control.

The library keeps browser defaults, page background, focus states, responsive spacing, and motion preferences consistent across the site.

Classless-first HTML

Raw headings, paragraphs, links, lists, blockquotes, buttons, inputs, code, and tables receive a Mercury baseline without extra classes.

<main>
  <h1>Semantic page title</h1>
  <p>Mercury styles plain HTML first. Add utilities only when the default rhythm needs a deliberate adjustment.</p>
  <button type="button">Continue</button>
</main>

System light mode

Mercury uses @media (prefers-color-scheme: light) to remap the core colour variables between light and dark mode depending on the users system preference.

Use the theme toggle in this guide to preview the same variable remap without changing your system settings. 
@media (prefers-color-scheme: light) {
  :root {
    --mc-color-bg: #efe7f4;
    --mc-color-accent: #17121c;
    --mc-color-muted: #5c5067;
    --mc-color-paper: #ffffff;
  }
}

Utilities

Display, flex, positioning, borders, overflow, and docs helpers

Utility API rules

Utilities use the mc- prefix, a property group, and a double-dash value. Responsive variants prepend the breakpoint and escaped colon in CSS, for example md:mc-d--grid.

PatternExampleMeaning
mc-{group}--{value}mc-p--5Set one property group directly.
mc-{axis}--{value}mc-mt--4Set one logical spacing axis.
{breakpoint}:mc-{group}--{value}lg:mc-grid--4Apply the utility from that breakpoint upward.
mc-{group}--automc-mx--autoUse keyword values where they are predictable.
